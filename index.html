重要提醒：关于跨域连接的问题 (CORS)

当您将网站部署到线上后，您的网站会拥有一个域名，而您尝试获取数据的 API 服务在另一个完全不同的域名上。此时，浏览器出于安全考虑，会阻止您的 JavaScript 代码直接获取数据，这就是跨域问题。

本应用默认通过一个公共代理服务器解决此问题。如果遇到网络连接错误，很可能是因为该代理服务器的临时访问权限已过期。

解决方法：

  1. 请访问 CORS-Anywhere 官方演示页面。
  2. 点击页面上的 "Request temporary access to the demo server" 按钮。
  3. 获取临时访问权限后，返回本页面刷新即可正常使用。

或者，您可以自行搭建 CORS 代理服务器，并相应修改 `app.js` 文件中的 `PROXY_URL` 变量。
